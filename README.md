# VLESS Shop Bot | Telegram-бот для продажи VPN

**VLESS Shop Bot** — комплексное решение для автоматизированной продажи VLESS‑конфигураций через Telegram с глубокой интеграцией в панель **3x‑ui** и собственной веб‑панелью управления.

## Возможности

- Полная автоматизация: от регистрации до выдачи ключа после оплаты
- Веб‑панель управления: сервера, тарифы, пользователи, платежи, логи
- Мульти‑хост: любое число серверов 3x‑ui, централизованное управление
- Гибкие тарифы: настройка месяцев и цены для каждого хоста
- Платежи: YooKassa, CryptoBot, Heleket, TON Connect, Telegram Stars
- Принудительная подписка на канал (опционально)
- Тестовый период (trial)
- Реферальная система (несколько типов начислений)
- Встроенная поддержка: внешний саппорт‑бот либо контакт пользователя
- Тест скорости по хостам: SSH‑speedtest + сетевая проверка

---

## Требования

1. Сервер Ubuntu/Debian с доступом по SSH
2. Домен, A‑запись которого указывает на IP сервера
3. Установленная панель 3x‑ui на одном или нескольких серверах

## Установка

Скрипт автоматически установит Docker, Nginx, Certbot, скачает и развернёт бота и панель.

1. Подключитесь по SSH к серверу

2. Выполните команду установки:

```bash
curl -sSL https://raw.githubusercontent.com/HolyLittleGirl/3xui-shopbot/main/install.sh | sudo bash
```

3. Следуйте инструкциям установщика:
   - Введите домен (например, `shop.example.com`)
   - Укажите email для SSL (Let's Encrypt)

4. После завершения вы получите доступ к веб-панели:
   ```
   Веб‑панель: https://your-domain.com/login
   Логин: admin
   Пароль: admin
   ```

## Первичная настройка

### 1. Вход в панель

Откройте `https://your-domain.com/login` и войдите под admin/admin. Рекомендуется сразу сменить пароль в разделе **Настройки → Настройки панели**.

### 2. Настройка Telegram

В разделе **Настройки → Основные** заполните:

- **Токен бота** — токен от [@BotFather](https://t.me/BotFather)
- **Имя телеграм бота** — username бота (например, `myshopbot`)
- **ID администратора в телеграмме** — ваш Telegram ID (можно узнать через [@userinfobot](https://t.me/userinfobot))

### 3. Добавление хоста 3x-ui

В разделе **Настройки → Управление хостами**:

1. Нажмите **Добавить хост**
2. Укажите:
   - **Название хоста** (произвольное)
   - **URL панели 3x-ui** (например, `https://panel.example.com`)
   - **Логин/пароль** от панели 3x-ui
   - **Inbound ID** — ID inbound в панели 3x-ui (обычно 1)

### 4. Создание тарифов

В разделе **Настройки → Тарифы**:

1. Выберите хост из списка
2. Добавьте тарифы:
   - **Название** (например, "1 месяц", "6 месяцев", "12 месяцев")
   - **Срок** (в месяцах)
   - **Цена в RUB**
   - **Цена в Telegram Stars** (опционально, например, 100 ⭐)

### 5. Запуск бота

После настройки нажмите **Запустить бота** в шапке панели.

## Настройка платёжных систем

Откройте **Настройки → Настройки платёжных систем** и заполните нужные поля.

### YooKassa

1. Укажите `yookassa_shop_id` и `yookassa_secret_key`
2. В кабинете YooKassa настройте URL вебхука:
   ```
   https://your-domain.com/yookassa-webhook
   ```
3. (Опционально) Укажите email для чеков

### CryptoBot

1. Получите токен в [@CryptoBot](https://t.me/CryptoBot) → Crypto Pay
2. Включите вебхуки в настройках CryptoBot:
   ```
   https://your-domain.com/cryptobot-webhook
   ```
3. Укажите `cryptobot_token` в настройках

### Heleket

1. Укажите `heleket_merchant_id` и `heleket_api_key`
2. Настройте вебхук в панели Heleket:
   ```
   https://your-domain.com/heleket-webhook
   ```

### TON Connect

1. Укажите `ton_wallet_address` — адрес кошелька для приёма платежей
2. Укажите `tonapi_key` — API ключ для получения курсов (опционально)

### Telegram Stars

**Не требует дополнительной настройки!** Telegram Stars доступен по умолчанию для всех ботов.

Для использования:
1. Укажите цену в звёздах при создании/редактировании тарифа
2. Пользователи смогут оплачивать ключи звёздами напрямую в боте

**Курс конвертации:** 1 звезда ≈ 1.25 RUB (используется для внутреннего учёта)

## Настройка тестов скорости (Speedtest)

Тесты скорости доступны из админ‑меню бота и из веб-панели.

### Подготовка хоста 3x-ui

В разделе **Настройки → Управление хостами** укажите параметры подключения к серверу 3x-ui:

- **SSH-хост** — адрес сервера (IP или домен)
- **SSH-порт** — порт (по умолчанию 22)
- **SSH-пользователь** — пользователь (например, `root`)
- **SSH-пароль** — пароль или оставьте пустым при использовании ключа
- **Путь к ключу** — путь к приватному ключу на сервере (опционально)

### Подготовка хоста ShopBot (панели шопа)

Для тестирования скорости с сервера, где развёрнута панель ShopBot:

1. В разделе **Настройки → Управление хостами** найдите ваш хост
2. Заполните SSH-параметры для этого хоста:
   - **SSH-хост** — адрес сервера ShopBot
   - **SSH-порт** — порт SSH (по умолчанию 22)
   - **SSH-пользователь** — пользователь (например, `root`)
   - **SSH-пароль** — пароль от сервера
3. Сохраните изменения

> **Примечание:** Убедитесь, что на сервере ShopBot установлен `speedtest-cli` или доступен `speedtest.net`.

### Запуск speedtest

- **Из бота:** Админ‑меню → Speedtest → выбрать хост или «Запустить для всех»
- **Из панели:** Кнопка «Run speedtests» на дашборде

Результаты сохраняются в БД и отображаются на дашборде у каждого хоста.

## Реферальная система

Настройки находятся в разделе **Настройки → Общие**.

### Типы начислений

1. **Процент с покупки реферала** — процент от суммы каждой покупки
2. **Фиксированная сумма за покупку** — фиксированная сумма за каждую покупку
3. **Фиксированный бонус при старте** — бонус пригласившему при первом запуске бота по реферальной ссылке

### Дополнительные параметры

- **Скидка для приглашённого** (%) — скидка на первую покупку
- **Минимальная сумма на вывод** — минимальная сумма для вывода средств (если реализовано)

### Реферальная ссылка

Формируется автоматически: `https://t.me/<bot_username>?start=ref_<telegram_id>`

Пользователи могут просмотреть свою ссылку и статистику в профиле.

## Настройка поддержки

В панели доступны два режима:

### 1. Внешний саппорт‑бот

- Укажите `support_bot_token` — токен бота поддержки
- Укажите `support_bot_username` — username бота
- Укажите `support_text` — приветственное сообщение

Пользователи переходят в отдельного бота по кнопке в разделе «Помощь».

### 2. Внешний контакт

- Укажите `support_user` — контакт поддержки (например, `@username`)

Кнопка ведёт в личные сообщения указанному контакту.

## Сохранение данных

**База данных хранится в Docker-томе** и сохраняется при перезапуске контейнера.

### Расположение данных

- **База данных:** Docker volume `shopbot-db` (путь в контейнере: `/app/project/data/users.db`)
- **Бэкапы:** `/app/project/data/backups/` (внутри контейнера)

### Проверка сохранения данных

Убедитесь, что данные сохраняются:

```bash
# Проверка состояния тома
docker volume ls | grep shopbot-db

# Просмотр информации о томе
docker volume inspect shopbot-db

# Проверка размера БД (должен расти при использовании)
docker exec 3xui-shopbot du -h /app/project/data/users.db
```

### Бэкап базы данных

**Автоматический бэкап** настраивается в веб-панели (раздел **Настройки → Бэкапы**).

**Ручной бэкап:**

```bash
# Копирование БД из контейнера
docker cp 3xui-shopbot:/app/project/data/users.db ./users.db.backup

# Или создание архива с бэкапом через панель управления
```

### Восстановление из бэкапа

```bash
# Остановка бота
docker-compose down

# Копирование бэкапа в том
docker cp ./users.db.backup 3xui-shopbot:/app/project/data/users.db

# Запуск бота
docker-compose up -d
```

### Важные замечания

1. **Не удаляйте Docker volume** `shopbot-db` — это приведёт к потере всех данных
2. При удалении проекта через `docker-compose down -v` тома также удаляются
3. Регулярно делайте бэкапы через веб-панель
4. При переезде на другой сервер экспортируйте том или скопируйте файл БД
